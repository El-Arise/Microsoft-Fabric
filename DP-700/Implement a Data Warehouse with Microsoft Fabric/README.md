📘 MICROSOFT FABRIC – Implement a Data Warehouse with Microsoft Fabric 

This folder contains my work for the Implement a Data Warehouse learning path in Microsoft Fabric. 
It is part of my training at the Poshem Technologies Institute DataChallenge, 
where I am learning how to design, load, secure, and manage data in a modern cloud‑based warehouse environment.

A data warehouse in Microsoft Fabric is a fully managed, scalable, SQL‑based analytics engine built on OneLake.

It supports full T‑SQL, dimensional modeling, secure data access, and enterprise‑grade monitoring.


This learning path introduces key parts of warehouse implementation in Microsoft Fabric.

🎯 LEARNING OBJECTIVES

1. Introduction to End‑to‑End Analytics with Microsoft Fabric
 
   Understanding how Fabric unifies data engineering, warehousing, real‑time analytics, and reporting in a single platform.
   
2. Get Started with Data Warehouses in Microsoft Fabric

   Learning the fundamentals of Fabric warehouses, including storage, compute, SQL endpoints, and workspace integration.

3. Load Data into a Fabric Data Warehouse

    Exploring ingestion methods such as:
    - Full (initial) loads
    - Incremental loads
    - Staging tables
    - COPY statement
    - Pipelines and the Copy Data Assistant.
      
4. Query a Fabric Data Warehouse
   
   Using T‑SQL, the Visual Query Editor, and semantic modeling tools to explore and analyze warehouse data.

5. Monitor a Fabric Data Warehouse

   Using Dynamic Management Views (DMVs), Query Insights, and the Capacity Metrics App to track performance, sessions, and long‑running queries.

6. Secure a Fabric Data Warehouse

   Implementing enterprise‑grade security features:
    - Workspace roles
    - Item‑level permissions
    - SQL permissions (GRANT, DENY, REVOKE)
    - Column‑Level Security (CLS)
    - Row‑Level Security (RLS)
    - Dynamic Data Masking (DDM)
